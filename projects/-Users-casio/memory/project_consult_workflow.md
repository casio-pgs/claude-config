ユーザーが「コンサル動画をテキスト化してObsidianに保存してWordPressにする」と依頼してきたときの定型フロー。

## フロー手順

1. **Premiere Proで文字起こし** → ユーザーが .txt を指定フォルダに保存
2. **Obsidianに議事録保存** → `/生徒/{名前}/コンサル議事録_{日付}.md`
3. **XMindマインドマップ作成** → Python+zipfileで `.xmind` ファイル生成（XMind 8形式）、同フォルダに保存
4. **WordPress記事作成** → 第三者向けノウハウ記事として `/生徒/{名前}/WordPress記事_{日付}.md` に作成
5. **WordPress投稿** → `13_投稿する記事/` にコピーして `obsidian_to_wp.py` の `post_to_wordpress()` を直接呼ぶ

## 使用スクリプト

- **WordPress投稿:** `~/obsidian-to-wp/obsidian_to_wp.py` の `post_to_wordpress(filepath)`
- **WP投稿先:** `https://casio-pgs.com`（予約投稿：翌日9:00〜）

## AIログ解説フォーマット（新シリーズ）

作業の記録を記事にするときは以下の5層構造で書く：

1. **STEP 1 ── プロンプト（依頼）** → 実際にユーザーが言った言葉をそのまま
2. **STEP 2 ── Claudeの判断プロセス** → 何を考えてどう方針を決めたか
3. **STEP 3 ── 実行したコード** → 実際に書いたコードをそのまま掲載
4. **STEP 4 ── ターミナルログ** → 実行結果の出力をそのまま掲載
5. **STEP 5 ── 何が起きたのかの解説** → コード・ログの意味を噛み砕いて説明

ファイル名プレフィックス：`AIログ解説_〇〇_日付.md`

## 記事作成ルール（Obsidianグラフ活用）

記事を作成する際は必ず以下を追加する：
1. **frontmatterにタグ**（5〜7個）: 記事の内容に合わせて設定
2. **末尾に「関連記事」セクション**で既存記事への`[[内部リンク]]`を2〜3件追加
3. `[[リンク]]`はWordPress投稿時にスクリプトが自動でWordPressのURLに変換してリンクになる
   - Obsidian内 → グラフビューのリンクとして機能
   - WordPress内 → クリッカブルなリンクとして機能

**⚠️ 関連記事リンク作成時の注意：**
- `[[ファイル名]]` はVaultにファイルが存在 **かつ** WordPressに投稿済みでないとリンクにならない
- 記事作成前に `https://casio-pgs.com/wp-json/wp/v2/posts?search=キーワード` で実在確認すること
- WordPressに存在しない記事は `[[リンク]]` ではなくプレーンテキストになってしまう
- **関連記事リンクは必須。投稿前に必ずWP検索して実在するスラッグで [[スラッグ|タイトル]] を末尾に2〜3件入れること。リンクなしで投稿しない。**

## 注意

- WhisperではなくPremiereProで文字起こしする（ユーザー希望）
- WordPress記事はコンサル内容をそのままではなく「第三者向けノウハウ記事」として書く
- XMindはMermaidではなく `.xmind` ファイルとして作成する

**Why:** 同じパターンが繰り返されるためフロー全体を自動化したい
**How to apply:** 「コンサル動画をテキストに」「さっきのフローで」と言われたらこのフローを実行する

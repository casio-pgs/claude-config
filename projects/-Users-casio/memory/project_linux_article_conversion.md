## 記事スタイルのルール

- **トーン：柔らかく・語りかける口調**（硬い技術文書にしない）
- 見出しは「〜する」「〜するには」など動詞系にする
- コードブロックにはコメントで説明を添える
- 表は必要な場合のみ（全部表にしない）
- 関係のない雑談・挨拶・無関係な質問は除外
- 末尾に `> **メモ**` で一言コメントを添える（実践的なアドバイスや「ここが大事」な一言）

### 良いトーンの例（mvコマンド記事より）

```
ターミナルでファイルを移動するときは `mv` コマンドを使います。「move」の略です。

移動と名前変更が同じコマンドというのが最初は少し不思議に感じるかもしれませんが、
「移動先の名前を変えている」と考えると自然です。

> **メモ**
> `mv` はシンプルだけど、間違えると元に戻せないので注意。
> まずは `ls` で確認してから実行する習慣をつけると失敗しにくいです。
```
